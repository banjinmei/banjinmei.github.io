img居中

img标签在HTML页面中居中显示，可以采用以下几种方法：

‌一、水平居中‌

‌使用text-align属性‌：

给img的父元素设置text-align: center;，这样img标签就会继承这个属性，实现水平居中的效果‌12。
‌使用margin属性‌：

给img标签设置margin-left: auto;和margin-right: auto;，并添加display: block;，这样img标签就会自动计算左右两边的空间，使得它在水平方向上居中‌12。
‌使用flex布局‌：

给img的父元素设置display: flex;和justify-content: center;，这样img标签就会在父元素的水平方向上居中‌12。

‌二、垂直居中‌

‌使用flex布局‌：

给包裹img的父元素设置**display: flex;和align-items: center;**，即可实现垂直居中‌45。
‌使用display: table;‌：

给父元素设置display: table;，给子元素（即包裹img的容器）设置display: table-cell;和vertical-align: middle;，即可实现垂直居中‌4。
‌使用绝对定位‌：

通过绝对定位的方式，结合top、left、transform等属性，也可以实现img的垂直居中，但这种方法需要更精细的布局控制‌4。
根据实际需求，可以选择适合的方法来实现img标签的居中显示。

NOW() 和 CURRENT_DATE() 的区别

NOW() 和 CURRENT_DATE() 的区别
1 返回值类型不同：
NOW() 返回的是 DATETIME 类型，包含日期和时间（精确到秒）。
CURRENT_DATE() 返回的是 DATE 类型，仅包含日期部分，不包含时间。
2 精度不同：
NOW() 的精度更高，适合用于需要精确时间戳的场合。
CURRENT_DATE() 只提供日期精度，适合在仅关注日期的场景下使用。
3 使用场景不同：
在需要完整时间戳的场合（如日志记录、事件跟踪、时间差计算），NOW() 是更好的选择。
在仅需关注日期的场合（如生日、发布日期、账单日期），CURRENT_DATE() 更加合适。
4 时间计算差异：
使用 NOW() 可以直接进行时间差计算，因为它包含了时间部分。
使用 CURRENT_DATE() 进行时间计算时，比较的只是日期部分，因此在计算时不考虑具体时间。

mysql 修改 累增

iuser表 UserEMail 为1@qq.com 累增 如2@qq.com 3@qq.com 4@qq.com 有多少用户则该多少个。批量修改 UserEMail 怎么写mysql
在 MySQL 中，你可以使用变量结合UPDATE语句来实现这个批量修改UserEMail字段值的需求。假设表名为iuser，示例代码如下：
sql
– 设置初始值为1
SET @num := 1;
– 更新表中的数据
UPDATE iuser
SET UserEMail = CONCAT(@num := @num + 1, ‘@qq.com’);
代码解释：
SET @num := 1;：定义了一个用户变量@num，并初始化为1。
UPDATE iuser SET UserEMail = CONCAT(@num := @num + 1, ‘@qq.com’);：在更新语句中，使用CONCAT函数把变量@num不断自增的值与‘@qq.com’拼接起来，再赋值给UserEMail字段。每次更新时，@num变量都会自动加 1，从而达成依次生成1@qq.com、2@qq.com 等邮箱地址的效果。
