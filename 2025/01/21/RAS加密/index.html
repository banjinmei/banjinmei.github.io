c# RAS

C#非对称加密 例子

字符串解密：

输入的密文和私钥都应该是合法的 Base64 编码字符串，否则 Convert.FromBase64String 会抛出异常。

私钥签名

data：需要签名的数据，这里是一个简单的字符串 “Hello, World!”。
privateKey：用于签名的私钥，以 Base64 编码的字符串形式提供。需要将 “your_private_key_here” 替换为实际的私钥。
SignData 函数：
byte[] dataBytes = Encoding.UTF8.GetBytes(data);：将需要签名的数据转换为字节数组，因为签名操作通常是在字节级别进行的。
byte[] privateKeyBytes = Convert.FromBase64String(privateKey);：将 Base64 编码的私钥转换为字节数组，以便 RSA 算法可以使用。
using (RSA rsa = RSA.Create())：创建一个 RSA 算法的实例。
rsa.ImportRSAPrivateKey(privateKeyBytes, out _);：将私钥导入 RSA 算法实例。
return rsa.SignData(dataBytes, HashAlgorithmName.SHA256, RSASignaturePadding.Pkcs1);：使用导入的私钥对数据进行签名。
dataBytes：要签名的数据的字节表示。
HashAlgorithmName.SHA256：使用 SHA256 算法对数据进行哈希处理。
RSASignaturePadding.Pkcs1：使用 PKCS #1 签名填充方案。

使用说明：

将 your_private_key_here 替换为自己的 RSA 私钥。
程序将对 “Hello, World!” 这个数据进行签名。
签名结果将以 Base64 编码的字符串形式输出。
运行程序，如果一切正常，将看到生成的签名；如果出现异常，将输出错误消息。
请确保私钥的安全性，因为私钥用于生成签名，一旦泄露，签名的安全性将受到威胁。同时，根据需要可以使用不同的哈希算法和签名填充方案，但要确保通信双方使用相同的设置，以保证签名验证的有效性。
此外，在实际应用中，可能需要考虑对签名结果进行存储或传输，以及对签名的验证过程，以确保数据的完整性和来源的真实性。以下是一个简单的签名验证示例：

data：原始数据，用于验证签名是否是对该数据的签名。
signatureBase64：需要验证的签名，以 Base64 编码的字符串形式提供。
publicKey：用于验证签名的公钥，以 Base64 编码的字符串形式提供。
VerifySignature 函数：
byte[] dataBytes = Encoding.UTF8.GetBytes(data);：将原始数据转换为字节数组。
byte[] signatureBytes = Convert.FromBase64String(signatureBase64);：将 Base64 编码的签名转换为字节数组。
byte[] publicKeyBytes = Convert.FromBase64String(publicKey);：将 Base64 编码的公钥转换为字节数组。
using (RSA rsa = RSA.Create())：创建一个 RSA 算法的实例。
rsa.ImportRSAPublicKey(publicKeyBytes, out _);：将公钥导入 RSA 算法实例。
return rsa.VerifyData(dataBytes, signatureBytes, HashAlgorithmName.SHA256, RSASignaturePadding.Pkcs1);：使用公钥验证签名。
dataBytes：原始数据的字节表示。
signatureBytes：要验证的签名的字节表示。
HashAlgorithmName.SHA256：使用 SHA256 哈希算法，与签名时使用的哈希算法保持一致。
RSASignaturePadding.Pkcs1：使用 PKCS #1 签名填充方案，与签名时使用的填充方案保持一致。

使用说明：

将 your_public_key_here 替换为自己的 RSA 公钥。
将 your_signature_here 替换为要验证的签名。
程序将对签名进行验证，并输出验证结果。
运行程序，如果签名有效，将输出 Signature is valid: true；否则，将输出 Signature is valid: false 或错误消息。
确保公钥和签名的完整性，以及使用与签名时相同的哈希算法和填充方案，这样才能正确验证签名的有效性。
