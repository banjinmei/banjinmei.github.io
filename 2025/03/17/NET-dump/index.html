dotnet-dump

dotnet-dump 是一个用于 收集和分析 .NET 进程内存转储 的工具，适用于 排查内存占用过高、GC 问题、内存泄漏 等问题。

3. 分析 dotnet 转储

步骤 1：进入交互模式

会进入 dotnet-dump 交互模式，类似这样：

dotnet-dump analyze /root/core_134609.dmp
Loading core dump…
(133920) loaded core dump from “/root/core_134609.dmp”
Type ‘help’ for more information.

步骤 2：分析内存占用
① 查看托管堆内存

会列出所有 .NET 对象 及其内存占用：

vbnet
MT Count TotalSize Class Name
00007ffb98b33800 10 2048 System.String
00007ffb98b3f8f8 5 1024 System.Byte[]
…
如果 TotalSize 非常大，可能是 大对象 (LOH) 或 GC 没有及时回收 导致的。
